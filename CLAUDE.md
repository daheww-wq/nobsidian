Obsidian의 유기적 백링크 + Notion의 구조적 블록 에디터를 결합.
v2에서 AI 요약/태그, 지식 그래프 추가.

## 4. 버전 범위

### v1 (MVP) — 핵심 기능

- GitHub OAuth 로그인 (Vercel httpOnly 쿠키)
- 레포 선택 및 파일 트리 로딩
- 폴더/노트 CRUD (생성/이름변경/삭제)
- @dnd-kit/sortable 드래그앤드롭 (순서 변경 + 폴더 이동)
- 우클릭 컨텍스트 메뉴 (아이템 오른쪽에 위치)
- **BlockNote** WYSIWYG 블록 에디터 (/ 슬래시 메뉴, 슬림 인라인 툴바)
- **기본 백링크**: `[[notename]]` 문법, 자동완성, 깨진 링크 표시, 사이드바 연결 노트 목록
- **검색**: 제목+본문+태그 실시간 검색, 날짜순/관련도순 정렬
- 이미지 드래그 업로드 → `/assets/images/`
- 자동 저장 (2초 디바운스 + 30초 배치 커밋) + 수동 저장 (Cmd+S)
- 수정 이력: GitHub Commits 기반 diff 뷰어 + 복원
- ⚠️ 오프라인 미지원 (온라인 전용)

### v2 — AI + 지식 그래프

- OpenAI API 기반 노트 요약 자동 생성
- 키워드 기반 자동 태그
- 이름 변경 시 백링크 자동 업데이트
- D3.js 지식 그래프 (태그 + 백링크 기반)
- 우측 Detail Panel 토글 (AI 요약, 태그, 연결 노트)

### v3 (로드맵)

- 팀 협업 (공유 레포 동시 편집)
- PWA + IndexedDB 오프라인 지원
- 모바일 반응형
- 다중 레포

## 6. 데이터 구조 요약

### 노트 파일 (마크다운 + frontmatter)

```yaml
---
id: uuid-v4
title: 노트 제목
created: ISO8601
updated: ISO8601
tags: [태그1, 태그2]
summary: 'AI 요약 (v2)'
---
본문 마크다운...
[[다른노트명]]  ← 백링크 (v1 기본 지원)
```

### 레포 구조

```
{repo}/
├── .notegraph/config.json
├── assets/images/
├── {폴더명}/_meta.json (선택)
├── {폴더명}/{노트}.md
└── {루트노트}.md
```

### BlockNote 저장 흐름

```
에디터 입력 (Block JSON)
  → 2초 디바운스 → 배치 큐 추가
  → 30초 배치 윈도우 종료 → blocknoteToMarkdown() 직렬화
  → GitHub API PUT (단일 배치 커밋: "[autosave] N changes")

수동 저장 (Cmd+S)
  → 배치 큐 즉시 플러시
  → GitHub API PUT ("[save]")

파일 열기
  → GitHub API GET (마크다운)
  → markdownToBlocks() 역직렬화
  → BlockNote 에디터에 로딩
```

### 클라이언트 상태 (보안 원칙)

```typescript
// ✅ 클라이언트 Zustand에 있어야 하는 것
interface AuthState {
  isAuthenticated: boolean;
  user: GitHubUser | null;
  // accessToken은 없음 — httpOnly 쿠키로만 관리
}

// ❌ 클라이언트 상태에 절대 없어야 하는 것
// accessToken: string  ← 서버사이드 httpOnly 쿠키 전용
```

## 7. 에이전트 작업 규칙

에이전트는 작업 전 반드시 아래를 확인한다:

1. **PLANNING.md** → 현재 구현 단계와 완료 여부
2. **docs/기능명세서.md** → 해당 기능의 예외/보안 케이스
3. **docs/AGENT.md** → 자신의 역할 정의와 순환 프로세스
4. **이 파일(CLAUDE.md)** → ADR과 OQ 중 작업에 영향을 미치는 것이 있는지

**코딩 원칙**

- 모든 컴포넌트: TypeScript strict 모드
- API Routes: 인증 미들웨어 적용 필수
- GitHub API 호출: 항상 에러 핸들링 포함
- 환경변수: `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`, `OPENAI_API_KEY`만 사용
- BlockNote 저장 시 반드시 `blocknoteToMarkdown()` 경유 (내부 JSON 직접 저장 금지)
- **클라이언트에 GitHub 토큰 절대 노출 금지** (ADR-22)
- 자동저장은 배치 큐 경유 (GitHub API 직접 호출 금지, ADR-20)
- `triggerSave` 내 fetch 시작 전 `savedPath = activePath` 캡처 필수 (ADR-25). 완료 후 `activePath !== savedPath`이면 `setSha`/`setSaveStatus` 호출 금지
- 아이콘은 반드시 `lucide-react`만 사용 (ADR-23). 이모지, 텍스트 기호, react-icons 혼용 금지
- `parseFrontmatter` 호출 시 경로(`path`) 인자 전달 필수 — 파일명 폴백 제목 추출에 필요 (ADR-26)
