## 1. 제품 개요

### 1.1 배경 및 목적

GitHub에 마크다운 원본을 보관하면서, Notion처럼 쉽게 편집하고 노트 간 연결을 관리하는 개인용 웹 지식 관리 도구.

Obsidian의 유기적 연결성(backlink, 계층 없는 지식 네트워크)과 Notion의 구조적 문서 관리(폴더/페이지 계층, 리치 에디터)를 결합한 웹 기반 앱. 파일은 **GitHub 레포지터리**에 마크다운(.md)으로 저장되어 버전 관리, 이식성을 확보한다.

> ⚠️ **v1은 온라인 전용이다.** GitHub API를 통해 파일이 레포에 저장되므로 이식성은 높지만, 오프라인 편집/저장은 v1에서 지원하지 않는다. 오프라인 지원(PWA + IndexedDB)은 v3 로드맵이다.

### 1.2 핵심 가치

| 가치      | 설명                                                                        |
| --------- | --------------------------------------------------------------------------- |
| 소유권    | 내 GitHub 레포 = 내 파일. 벤더 종속 없음. 서비스가 종료되어도 파일은 남는다 |
| 연결성    | `[[백링크]]` 기반 노트 간 연결 (v1), 지식 그래프 시각화 (v2)                |
| 편집 경험 | BlockNote 기반 Notion 스타일 블록 에디터 + / 슬래시 메뉴                    |
| 탐색성    | 제목+본문+태그 통합 실시간 검색 (v1)                                        |
| 자동화    | AI 요약/태그로 메타데이터 자동 생성 (v2)                                    |

## 2. 사용자 정의

### 2.1 주요 사용자 (Primary Persona)

**개인 지식 관리(PKM) 사용자**

- 마크다운에 익숙하고 파일 소유권을 중시하는 개발자/연구자/기획자
- Obsidian은 쓰지만 웹 접근성이 아쉬웠던 사용자
- Notion을 쓰지만 데이터 종속이 불안한 사용자
- 기존 마크다운 폴더를 GitHub에 올려 사용 중인 사용자

### 2.2 제외 대상 (v1 기준)

- 팀 협업 중심 사용자 (v3+ 고려)
- 오프라인 전용 사용자 (v3에서 PWA 지원)
- GitHub 계정이 없는 일반 사용자

## 4. 기능 요구사항 요약

### 4.1 인증 및 저장소 연동

- GitHub OAuth 2.0 로그인
- 연결 시 요청하는 레포 권한 범위(`repo` scope) 명시 온보딩
- 사용자 레포 목록 조회 및 선택
- 선택된 레포를 `localStorage` + GitHub API로 유지
- 기존 마크다운 폴더 가져오기: frontmatter 없는 `.md` 파일 허용
- 세션 만료 시 자동 재인증 흐름

### 4.2 사이드바 (파일 트리)

- 루트 레벨: 폴더/노트 혼합 표시
- 폴더는 접기/펼치기 가능 (중첩 무제한)
- 우클릭 컨텍스트 메뉴: 우클릭한 아이템 **바로 오른쪽**에 표시 (좌표 자동 반전)
- 드래그앤드롭 (`@dnd-kit/sortable`):
  - 아이템 사이 드롭 → 같은 레벨 순서 변경 (2px 녹색 선 표시)
  - 폴더 위 800ms 호버 → 폴더 열림, 드롭 시 하위로 이동 (녹색 하이라이트)
  - 폴더 드래그 → 하위 파일 전체 경로 일괄 변경
- **사이드바 하단**: 현재 열린 노트를 참조하는 연결 노트 목록 (v1)

### 4.3 검색 (v1 신규)

- 사이드바 상단 검색 입력창 (항상 표시)
- 실시간 검색: 입력할 때마다 결과 갱신
- 검색 범위: 노트 제목 + 본문 텍스트 + frontmatter 태그
- 빈 쿼리 시: 최근 수정 노트 목록 표시
- 결과 정렬: 관련도순 (기본) / 날짜순 토글
- 결과 클릭: 해당 노트 열기 + 검색어 하이라이트
- 결과 없음 상태: "결과 없음" + 새 노트 생성 제안

### 4.4 에디터

- **BlockNote** 기반 WYSIWYG 블록 에디터 (TipTap/ProseMirror 위에 구축)
- 블록 삽입: `/` 슬래시 메뉴 — H2/H3, 체크리스트, 표, 코드블록, 이미지, 백링크
- **인라인 툴바**: 텍스트 드래그 선택 시 선택 영역 위에 플로팅 표시
  - 구성: Bold / Italic / Strikethrough / InlineCode / Link (5버튼)
  - 스타일: 흰 배경, 회색 테두리, 그림자, 선택 영역 위 중앙 정렬
- 에디터 본문 최대 너비: 960px
- v1: WYSIWYG 전용 (마크다운 토글 없음)
- 이미지: 드래그/붙여넣기 업로드 → GitHub 레포 `/assets/images/` 저장

### 4.5 백링크 (v1 기본)

- `[[notename]]` 문법: BlockNote 커스텀 인라인 노드
- `[[` 입력 시 자동완성 드롭다운 (노트 목록 퍼지 매칭)
- 깨진 링크: 존재하지 않는 노트명은 회색+취소선으로 표시
- 연결 노트 목록: 현재 노트를 참조하는 노트들을 사이드바 하단에 표시 (v1)
- 백링크 클릭: 해당 노트로 이동

### 4.6 저장 (배치 커밋)

- 자동 저장: 2초 디바운스 후 배치 큐에 추가 → 30초 윈도우 종료 시 단일 커밋 (`[autosave] N changes`)
- 수동 저장: Cmd/Ctrl+S → 배치 큐 즉시 플러시 → 단독 커밋 (`[save]`)
- SHA 충돌 감지 및 머지 UI
- 오프라인 감지: 로컬 임시 저장 → 온라인 복구 시 자동 커밋
- 저장 상태 표시: Saved / Saving... / Unsaved changes / Error

### 4.7 수정 이력

- GitHub Commits를 이력으로 활용 (별도 파일 불필요)
- 에디터 툴바 "이력 보기" 버튼으로 진입
- diff 뷰: 추가(녹색) / 삭제(빨간색) / 컨텍스트(회색)
- 버전 복원: 선택 커밋의 내용으로 현재 파일 덮어쓰기 (새 commit: `[restore]`)

### 4.8 AI 기능 (v2)

- 노트 저장 후 백그라운드 OpenAI 호출
- 요약: frontmatter `summary:` 필드에 삽입
- 태그: frontmatter `tags:` 배열에 자동 추가 (기존 태그 보존)
- 사용자 수동 트리거 버튼 제공

### 4.9 지식 그래프 (v2)

- D3.js force-directed graph
- 노드: 노트, 엣지: 백링크 + 공유 태그
- 클릭 시 해당 노트로 이동
- 필터: 태그별, 폴더별

## 5. 비기능 요구사항

### 5.1 성능

| 항목             | 목표                               |
| ---------------- | ---------------------------------- |
| 사이드바 로딩    | < 1.5초 (레포 100개 파일 기준)     |
| 에디터 입력 지연 | < 50ms                             |
| 검색 응답        | < 200ms (클라이언트 인덱스 기준)   |
| 자동 저장 응답   | < 3초 (GitHub API round-trip 포함) |
| 지식 그래프 렌더 | < 2초 (노트 500개 기준)            |

### 5.2 보안

- GitHub OAuth token: `httpOnly` 쿠키만 (클라이언트 Zustand 상태에 토큰 없음)
- 클라이언트 상태에는 `isAuthenticated: boolean` + `user` 정보만 보관
- API 키(OpenAI): 서버사이드 환경변수만 허용, 클라이언트 노출 금지
- GitHub API 권한: `repo` scope만 요청 (최소 권한 원칙, 사용자에게 명시)
- CSRF 방어: state 파라미터 검증

### 5.3 호환성

- 브라우저: Chrome 120+, Firefox 120+, Safari 17+
- 반응형: 데스크탑(1280px+) 우선, 태블릿(768px+) 지원
- 모바일: v3 로드맵
- ⚠️ 오프라인: v1 미지원 (온라인 전용)
